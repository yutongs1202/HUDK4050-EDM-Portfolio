Machine Learning and Classification¶

Classification¶

In the machine learning world, sometimes "teachers" need to prepare the training experiences for the computer program : We need to clearly tell the computer program what is "right" and what is "wrong" (and yes, sometimes we don't need to do that). This approach is generally referred as supervised learning. Classification is an area of supervised machine learning that tries to predict which class or category some entity belongs to, based on its features.

Similar to how one studies for a test by completing endless practice problems, the algorithm learns how to properly identify or classify a given input by processing large, labeled datasets until it can do so with an elevated level of confidencethe larger the training set, containing rich and diverse examples, the more accurate the algorithm.

Depending on outcome, we can have binary classificaiton, where the predicted variable is a dichotomous, and multiclass or multinomial classificaition, where the variable can be a categorical variable containing multiple levels.

Logistic Regression¶

Logistic regression is probably the most fundamental classification method. As suggested by its name, it belongs to the regression family and is related to the linear regression we have seen. Logistic regression is fast and relatively uncomplicated, and its convenient for you to interpret the results.

You probably have noticed that the results of two models are not identical. And you are absolutely correct. sklearn and statsmodels set the default logistic regression a little bit differently. The most obvious one is statsmodels does not estimate the intercept by default. Therefore, the results of all coefficient will be shifted. There are more differences such as the regularization parameter. See a discussion here on StackExchange: https://stats.stackexchange.com/questions/203740/logistic-regression-scikit-learn-vs-statsmodels

And here are the LogisticRegression documentation in sklearn and the Logit documentation in statsmodles. You may also want to read in detail if you are curious about what is happening under the hood.

But, does it matter if we get different results? Well... yes and no.

First, if your goal is to evaluate the impact of each predictor on the predicted variable, then yes. The implementation in both sklearn and statsmodels deviate a little bit from the most standard way of logistic regression. So if you want to interpret the result like what you would do in a stats course, then you probably want to spend some time to get into the weeds.

However, if you goal is simply building a classifier, any of these two implementation will do. Both models (i.e., moocLogitModel1 and moocLogitModel2) will be able to predict the certified_yes with given features with no problems. We will check their performance at a later time.

Decision Tree¶

A Decision Tree is another supervised Machine learning algorithm. The decision tree is like a tree with nodes. The branches depend on a number of factors. It splits data into branches like these till it achieves a threshold value. A decision tree consists of the root nodes, children nodes, and leaf nodes.

Decision trees learn from data to approximate the training data with a set of if-then-else decision rules. The deeper the tree, the more complex the decision rules and the fitter the model.

Decision tree has various algorithms to implement. In sklearn, an optimised version of CART algorithm is used.

Naive Bayes¶

Naive Bayes models are a group of extremely fast and simple classification algorithms that are often suitable for very high-dimensional datasets. Because they are so fast and have so few tunable parameters, they end up being very useful as a quick-and-dirty baseline for a classification problem.

Naive Bayes classifiers are built on Bayesian classification methods. These rely on Bayes's theorem, which is an equation describing the relationship of conditional probabilities of statistical quantities. In Bayesian classification, we're interested in finding the probability of a label given some observed features.

If we are trying to decide between two labelslet's call them L1 and L2then one way to make this decision is to compute the ratio of the posterior probabilities for each label.

All we need now is some model by which we can compute for each label. Such a model is called a generative model because it specifies the hypothetical random process that generates the data. Specifying this generative model for each label is the main piece of the training of such a Bayesian classifier. The general version of such a training step is a very difficult task, but we can make it simpler through the use of some simplifying assumptions about the form of this model.

This is where the "naive" in "naive Bayes" comes in: if we make very naive assumptions about the generative model for each label, we can find a rough approximation of the generative model for each class, and then proceed with the Bayesian classification.

Model Evaluation¶

As discussed previously, we need a sample of data held back from training your model that is used to give an estimate of model performance.

In the machine learning world, this type of data is often called either validation data or test data as opposed to the training data, which was used to train the model.

At the basic level, our model is pretty simple and straightfoward. So there is no point to differentiate the validation data and the test data. In 4050, we will probably use these terms somewhat interchangeably. But notice that they are different. Read more here: https://www.applause.com/blog/training-data-validation-data-vs-test-data

We will talk about some more details of model evaluation in the next couple weeks. But let me introduce you to a simplistic way of knowing the model performance -- confusion matrix.

It is a performance measurement for machine learning classification problem where output can be two or more classes. It is a table with 4 different combinations of predicted and actual values.

True Positive: You predicted positive and its true. You predicted that a person has covid and he/she actually has.

True Negative: Interpretation: You predicted negative and its true. You predicted that a person does not have covid and he/she actually does not have it.

False Positive: (Type 1 Error) You predicted positive but its false. You predicted that a person has covid, but he/she actually does not have it.

False Negative: (Type 2 Error) You predicted negative but its true. You predicted that a person does not have covid, but he/she actually has it.

Once we have these four numbers, things become pretty easy. The simplistic measure would be accurracy. It basically tells us how often the classifier is correct.

Here are a couple of things we need to do:

1.Split the data into two sets (by convention it is 80:20, but it can be other ratio -- either larger or smaller depending on how many data points you have).

2.Re-train the models

3.Feed the Xs in the testing dataset and obtain the predicted Ys.

4.Compare the predicted Ys with what is actually in the testing dataset (the ground truth) and obtain the confusion matrix.

5.Obtain the accuracy from the confusion matrix.
